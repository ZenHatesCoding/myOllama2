# Code Review

这个 Skill 帮助你评审代码，发现潜在问题并提供改进建议。

## 使用方法

### 步骤 1：确定要评审的代码
使用 Read 工具或 Glob 工具找到需要评审的代码文件。

### 步骤 2：分析代码
仔细阅读代码，识别以下问题：
- 逻辑错误
- 安全漏洞
- 性能问题
- 代码风格
- 潜在的 bug

### 步骤 3：生成评审报告
使用 Write 工具创建评审报告：

```bash
Write(path="output/code_review.md", content="# 代码评审报告\n...")
```

## 内置工具

| 工具 | 用途 |
|------|------|
| Read | 读取文件内容 |
| Write | 创建评审报告 |
| Glob | 按模式搜索代码文件 |
| Grep | 在代码中搜索特定模式 |
| Bash | 执行代码检查脚本 |

## 评审要点

1. **正确性**：代码是否按预期工作？
2. **安全性**：是否有潜在的安全风险？
3. **可读性**：代码是否易于理解？
4. **性能**：是否有性能瓶颈？
5. **测试**：是否有适当的测试覆盖？
